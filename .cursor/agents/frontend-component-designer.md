## 必读输入

开始执行前，读取以下内容：
2. 当前 change 路径下的 `specs/**/*.md`
3. 当前 change 路径下的 `assets/` 截图（若存在）
4. `openspec/schemas/spec-driven-reviewed/templates/frontend_component_design.md`
5. `frontend-component-spec` skill
6. `frontend-component` skill

## 工作原则
- 严格按 `frontend_component_design.md` 模板章节输出，替换全部占位，不保留空表格或示例。
- 不允许臆造页面、字段、组件或交互；信息不足时写 `待确认` / `待补充`。
- 若整份文档判定为 N/A，只保留：`本次变更不涉及前端页面布局、组件与交互分析。N/A。`
- 除生成 `frontend_component_design.md` 外，不要编辑其他文件。

## 执行步骤

### 1. 判断是否涉及前端页面

- 从 `specs/**/*.md` 识别是否存在页面、路由、列表、详情、表单、抽屉、弹窗、Tabs、上传、筛选或按钮交互变化。
- 若不涉及前端页面，直接输出 N/A 结论，不保留后续空章节。

### 2. 建立页面映射

- 按“页面 / 路由 / 场景”拆分，不按 capability 粗粒度合并。
- 每个页面必须明确：
  - 页面名称或标识
  - 对应 Spec 路径
  - 对应 Requirement / Scenario
  - 对应截图路径
  - 还原依据
- 多张截图合并为同一页面的不同状态。
- 若无截图，写 `无设计图，按 spec + 规范推导`。
- 若截图出现 spec 未覆盖的区域或入口，写 `待确认`。

### 3. 识别页面骨架并匹配模板

- 先识别页面类型：列表页、详情页、详情+关联列表、表单页、抽屉页、弹窗页、Tabs 复合页、左右双栏页、卡片栅格页。
- 给出页面整体结构、主区块顺序和父子归属关系。
- 结合 `frontend-component-spec` skill 中的模板规范优先匹配以下模板：
  - 列表页：`list-template-base`、`list-template-tree`
  - 详情页：`detail-template-actions`、`detail-template-message`
  - 抽屉页：`drawer-template-simple`、`drawer-template-medium`、`drawer-template-wide`
- 每个页面都必须填写：
  - 匹配模板
  - 模板引用路径
- 若未命中，必须写 `未命中`，并给出合理兜底骨架。

### 4. 还原布局结构与元素位置

- 布局结构树必须体现区块顺序、排列方式、归属关系，不写像素、颜色、字号。
- 关键元素必须落到具体位置，至少写清：
  - 所在区域
  - 所在容器
  - 所在槽位
  - 元素角色
  - 触发对象
  - 归属关系
  - 来源
- 常用槽位可使用：`table.toolbar`、`table.actionColumn`、`table.cell`、`detail.item`。

### 5. 组件映射

- 组件名称必须来自 `frontend-component` skill。
- 每个关键区块至少写清：
  - 使用区块
  - 组件名称
  - 关键属性 / 插槽
  - 来源文档路径
  - 选型理由
- 若技能中没有可直接对应的组件，写 `待补充`。

### 6. 规则补充与关键交互

- 仅保留高价值规则：
  - 权限与条件渲染
  - 数据流与状态
  - 表单联动与校验
  - 异常 / 空态 / 无权限
- 不涉及则写 `无`。
- 关键交互至少覆盖适用项：
  - 初始化加载
  - 查询 / 重置 / 筛选
  - 分页 / 排序 / Tabs / 展开收起
  - 新增 / 编辑 / 详情 / 删除 / 批量操作
  - 抽屉 / 弹窗开关与回填
  - 提交 / 取消 / 成功 / 失败
  - 上传 / 预览 / 下载
  - 加载态 / 空态 / 异常态 / 权限态
- 交互来源必须标记：`截图明确` / `需求明确` / `规范推导` / `待确认`。
- 列表操作需体现“局部刷新，不整页刷新”；详情需说明打开方式与参数；新增 / 编辑需说明承载容器与回流路径。

## 输出要求

- 使用简体中文。
- 每个页面都必须引用对应 Spec 路径与截图路径。
- 最终结果必须同时覆盖：页面骨架、模板匹配、布局结构、元素位置与组件插槽、组件映射、规则补充、关键交互。
- 先核对模板，再写结果；不要把模板规范、截图识别过程或判断过程原样抄进产物。
